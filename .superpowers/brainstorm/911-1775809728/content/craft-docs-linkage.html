Docs — nmap (linked from Craft)
-sS Scan Type
TCP SYN Stealth Scan
The default and most popular scan. Sends a SYN packet (as if opening a connection) and waits for a response. SYN/ACK = open port, RST = closed. Never completes the TCP handshake, making it harder to detect in logs. Requires root/admin privileges. Faster than full TCP connect scan (-sT).
Example:
nmap -sS 192.168.1.1
⚠ Requires root/sudo privileges
-O Option
OS Detection
Enables OS fingerprinting. Nmap sends TCP/UDP probes and analyzes responses to guess the target's operating system. Compares against a database of ~5,000 known OS fingerprints. Results include OS family, version, generation, and device type. Accuracy varies — works best when at least one open and one closed port are found.
Example:
nmap -O --osscan-guess 192.168.1.1
⚠ Requires root/sudo privileges
--script vuln NSE Script
Vulnerability Scanning Scripts
Runs all scripts in the "vuln" category from Nmap Scripting Engine (NSE). Checks for known vulnerabilities including Heartbleed, SMB exploits, shellshock, SSL/TLS issues, and more. Each script tests for specific CVEs and reports findings. Can be slow on large scans — consider targeting specific ports. Use --script=vulners for an alternative CVE lookup approach.
Example:
nmap --script vuln -p 80,443 192.168.1.1
💡 Tip: Combine with -sV for better script detection

Link Flow

  1. User is in Craft → nmap, sees ⓘ next to each toggle
  2. Clicks ⓘ next to "-O OS Detection"
  3. App switches to Docs → nmap Reference and scrolls to the -O entry
  4. User reads verbose docs, then clicks ← Back to Craft button at the top
  5. Returns to Craft with all their selections preserved
