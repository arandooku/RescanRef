ScanRef — 3 Approaches

Single offline HTML file for shell/terminal command reference with search

A

Scroll Page + Sticky Search

One long scrollable page. Sections are headings with command blocks underneath. A sticky search bar at the top filters commands in real-time. Sidebar nav links jump to sections.

Pros

  • Simplest to build and maintain
  • Easy to edit — just add HTML blocks
  • Familiar pattern (like a cheatsheet)
  • Ctrl+F also works as fallback

Cons

  • Can feel long if sections grow
  • Less interactive feel
B

Sidebar Tabs + Search

Fixed sidebar listing tool categories (Git, Docker, etc.). Clicking a category shows only that section's commands in the main area. Search bar filters across all sections and highlights matches.

Pros

  • Clean — only shows one section at a time
  • Sidebar gives quick overview of all categories
  • Scales better if sections grow

Cons

  • Slightly more JS complexity
  • Can't see commands across sections at once (except via search)
C

Collapsible Accordion + Search

All sections visible as collapsed headers. Click to expand/collapse. Search expands matching sections automatically. All on one page but compact.

Pros

  • Compact overview — see all categories at a glance
  • Expand only what you need
  • Middle ground between A and B

Cons

  • Extra clicks to expand sections
  • State resets on page reload (no persistence)

My Recommendation: B — Sidebar Tabs

For a command reference app you'll open daily, the sidebar gives you instant orientation — you can see all your categories and jump to any one. Search covers the "find anything fast" use case. It's the most app-like experience while still being a single HTML file. With under 50 commands and 5-10 sections, the sidebar won't feel overwhelming.
